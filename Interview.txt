Interview Questions & Answers


Q1. Why use MySQL + MongoDB together?
Answer
MySQL handles relational structured data.

MongoDB handles flexible schema data.

Using both improves scalability and performance.


Q2. Why Redis?
Answer
Redis is an in-memory database.

Used for:
- caching
- queues
- sessions
- faster response times


Q3. Why Worker Services?
Answer
Workers process heavy tasks in background.

Prevents API blocking.

Improves scalability.


Q4. What is BullMQ?
Answer
BullMQ is a Redis-based queue library.

Used for:
- background jobs
- retries
- delayed jobs
- async processing


Q5. What is a Dead Letter Queue?
Answer
Failed jobs are moved into DLQ.

Prevents data loss.

Helps retry failed operations.


Q6. Why use JWT?
Answer
JWT provides stateless authentication.

Scalable for distributed systems.
