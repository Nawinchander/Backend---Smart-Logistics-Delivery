Dead Letter Queue (DLQ)Why DLQ?If a job continuously fails:Network issueDB issueExternal API issueInstead of losing data:Send failed job to DLQRetry laterPrevent system crashdeadLetterQueue.jsconst { Queue } = require('bullmq');const Redis = require('ioredis');const connection = new Redis();const deadLetterQueue = new Queue('deadLetterQueue', {  connection,});module.exports = deadLetterQueue;Failed Job Handlerworker.on('failed', async (job, err) => {  await deadLetterQueue.add('failed-order', {    jobData: job.data,    reason: err.message,  });});12. Interview Questions & AnswersQ1. Why use MySQL + MongoDB together?Answer:MySQL handles relational structured data.MongoDB handles flexible schema data.Hybrid DB improves scalability and flexibility.Q2. Why use Redis?Answer:In-memory cache.Reduces DB calls.Improves API speed.Used for session management and queues.Q3. What is BullMQ?Answer:Queue library using Redis.Used for background processing.Supports retries, concurrency, scheduling.Q4. What is a Dead Letter Queue?Answer:Stores failed jobs.Prevents data loss.Helps debugging and retrying.Q5. Why use Worker Services?Answer:Moves heavy tasks away from APIs.Prevents API blocking.Improves scalability.Q6. What is Horizontal Scaling?Answer:Increasing number of servers.Stateless APIs enable horizontal scaling.Q7. Why JWT?Answer:Stateless authentication.Scalable.Easy microservice integration.Q8. Why Rate Limiting?Answer:Prevent brute force attacks.Protects APIs from abuse.Improves reliability.21.
